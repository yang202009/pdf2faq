# PDF2FAQ

将 PDF、书籍、报告或长文档转成**学习型 FAQ** 的 Claude Skill。

它不是做摘要，而是把文档重组为一份**覆盖优先、教练式、可追溯**的问答文档，帮助用户真正理解内容框架、关键论证、证据强弱、应用边界，以及哪些地方其实证据不足。

## 这个 Skill 能做什么

- 把 PDF / TXT / MD 文档转成结构化学习 FAQ
- 明确区分**原始文档**与**实际分析对象**
- 在输出中保留**来源关系、提取方式、锚点可靠性、文本抽取质量**
- 区分：**原文明确说明 / 原文暗示但未展开 / 综合推断 / 原文未说明 / 证据不足**
- 先生成**分析提纲**，确认方向后再生成完整 FAQ
- 根据文档质量选择：
  - **文档生产器**：适合结构清晰、可做完整 FAQ 的材料
  - **分析器**：适合 OCR 弱、锚点不稳、结构破损的材料
- 支持两种输出模式：
  - **标准学习版**
  - **商业审计增强版**

## 适用场景

适合这类请求：

- 帮我真正理解这本书/报告
- 把这个 PDF 转成学习 FAQ
- 提炼这份文档的核心框架和关键论点
- 帮我看哪些结论可靠，哪些其实只是推断
- 把这篇长文整理成可复用的学习笔记
- 我要做写作 / 研究 / 决策，先帮我拆清楚这份材料

## 工作方式

这个 Skill 的默认流程是：

1. 收集输入信息：
   - 原始文档路径
   - 实际分析对象路径（可选）
   - 阅读目标
   - 输出模式（可选）
2. 判断来源链路与提取质量
3. 识别文档类型、锚点策略、可靠性
4. 先生成**分析提纲**
5. 等用户确认
6. 再生成最终 Markdown FAQ

默认**不跳过提纲阶段**，除非用户明确要求一轮直出。

## 输入要求

### 必填

- **原始文档路径**
- **一句阅读目标**

### 强烈建议补充

- **实际分析对象路径**：如果你实际分析的是从 PDF 提取出来的 TXT / MD，请显式说明
- **输出模式**：标准学习版 / 商业审计增强版

### 默认规则

如果只给了一个路径，默认把它同时视为：

- 原始文档
- 实际分析对象

除非上下文已经明确是 PDF → TXT / OCR 的链路。

## 支持的输入形态

- 直接分析 PDF
- 分析由 PDF 提取出的 TXT / MD
- Mixed 模式：PDF 作为原始文档，TXT / MD 作为主要分析对象

这个 Skill 的一个核心要求是：

- **原始文档** = 用户真正想分析的源文件
- **实际分析对象** = 本次实际读取和推理所依据的文件

两者必须分开说明，不能混写。

## 输出模式说明

### 1. 标准学习版

适合：

- 深度理解
- 写作准备
- 学习笔记
- 搭建自己的知识框架

特点：

- 可读性更强
- 仍然保留来源与证据说明
- 有“如果继续想下去”与“进一步思考”
- 更适合个人学习与内容提炼

### 2. 商业审计增强版

适合：

- 需要更强溯源性
- 要交付给客户 / 团队 / 上级
- 更关注证据块、限制披露与审计摘要

特点：

- 额外包含文档处理说明
- 明确输入来源与完整性声明
- 更强调风险、边界和不确定性
- 更适合正式交付场景

## 目录结构

```text
pdf2faq/
├─ SKILL.md
├─ README.md
└─ references/
   ├─ analysis-outline-template.md
   ├─ final-faq-template.md
   ├─ final-faq-audit-template.md
   ├─ source-integrity-rules.md
   └─ index.md
```

其中：

- `SKILL.md`：主技能说明与工作流
- `references/analysis-outline-template.md`：分析提纲模板
- `references/final-faq-template.md`：标准学习版模板
- `references/final-faq-audit-template.md`：商业审计增强版模板
- `references/source-integrity-rules.md`：锚点可靠性中/低时必须加载的约束规则
- `references/index.md`：引用文件索引

## 安装方法

将整个 `pdf2faq` 文件夹复制到你的 Claude Code skills 目录中：

```text
~/.claude/skills/pdf2faq/
```

Windows 通常对应：

```text
C:\Users\你的用户名\.claude\skills\pdf2faq\
```

注意：**不要只复制 `SKILL.md`，必须连 `references/` 一起带上。**

## 使用方法

### 最简用法

直接告诉 Claude：

```text
原始文档路径 + 阅读目标
```

例如：

```text
D:\documents\book.pdf，帮我真正理解这本书的核心框架
```

### 推荐用法

在信息更完整时，建议这样给：

```text
原始文档路径：D:\documents\book.pdf
实际分析对象路径：D:\documents\book.txt
阅读目标：提炼核心框架，并区分哪些结论是原文明说、哪些只是推断
输出模式：标准学习版
```

### 商业审计增强版示例

```text
原始文档路径：D:\reports\industry-report.pdf
实际分析对象路径：D:\reports\industry-report-ocr.md
阅读目标：重点判断这份报告的假设、证据质量和商业含义
输出模式：商业审计增强版
```

## 使用时会发生什么

当 skill 被触发后，Claude 会按这个逻辑工作：

1. 读取原始文档与实际分析对象
2. 判断它们之间的来源关系
3. 检查提取质量、结构完整性、锚点可靠性
4. 判断更适合用“文档生产器”还是“分析器”
5. 先给出一份**分析提纲**
6. 在你确认后，生成完整学习 FAQ

如果锚点可靠性为**中**或**低**，或者分析对象是衍生 TXT / MD，skill 会强制加载 `source-integrity-rules.md`，避免假装有不存在的页码精度或来源可信度。

## 输出结果会包含什么

典型输出会包含：

- 文档处理说明
- 原始文档 / 实际分析对象说明
- 来源关系与提取方式
- 锚点可靠性与提取质量判断
- 3 分钟速读版
- 阅读诊断
- 阅读路线图（结构清晰时）
- 核心 FAQ
- 概念拆解
- 论证审查
- 应用迁移
- 反向提问
- 未说明 / 证据不足清单
- 一句话总结

在商业审计增强版中，还会额外突出：

- 输入来源与完整性声明
- 风险清单
- 适用边界 / 不适用边界
- 审计摘要

## 适合的文档类型

- 书籍
- 行业报告
- 研究报告
- 方法论文档
- 混合型长文档

并且会根据文档类型自动调整重点，例如：

- **书籍**：章节推进、作者世界观、核心模型
- **行业报告**：数据定义、趋势逻辑、商业含义、偏差风险
- **研究报告**：主张、证据质量、方法透明度、限制
- **方法论文档**：流程步骤、框架结构、适用场景、失效条件

## 质量原则

这个 skill 特别强调以下几点：

- 不把章节摘要伪装成 FAQ
- 不伪造页码精度
- 不伪造 provenance
- 不把推断说成原文明说
- 文档脏、OCR 差、结构碎时要**诚实降级**
- “进一步思考”不能写成空洞拔高或装深刻

## 常见提醒

- 如果 PDF 抽取质量差，FAQ 数量会主动缩减
- 如果图表、脚注、版式在线索中丢失，输出会明确披露
- 如果结构边界不清晰，阅读路线图会被降级或省略
- 如果你要正式交付给别人，优先用**商业审计增强版**
